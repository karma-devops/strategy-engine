1. Master — Core System Flow

TLDR: FastAPI app (main.py) boots on :8792, loads .env config, initializes SQLite DB, seeds operator user + default fleet, registers all routers. The InstanceManager singleton runs engine threads that poll HL market data, evaluate strategy signals, and execute trades via the exchange client. A standalone worker (scripts/worker.py) can run independently on :9999.

Dry-Run Architecture (Critical)

LayerVariableDefaultEffect
Global (.env)DRY_RUNfalseServer always connected to HL. Live account.
Per-UserUser.default_dry_runtrueNew instances seeded as paper trading.
Per-InstanceInstance.dry_runtrue (from user)Engine respects this via HyperLiquidClient. UI toggle controls it.
Override Flowget_hyperliquid_client(instance) always passes instance.dry_run to HyperLiquidClient — even when using global env credentials.

2. Database — 20 Tables (SQLite)

User API keys (PULS-R format) in users table. All schema migrations handled by idempotent _migrate_columns() on startup.
