About PULS-R

PULS-R is a strategy execution engine for HyperLiquid perpetuals markets. It translates PineScript trading strategies into Python, backtests them against historical candle data, and executes them live with precise position sizing and risk management.

How It Works

The engine runs in a continuous loop: fetch candles, run strategy, check signals, sync positions from HyperLiquid (source of truth), and execute trades only when the desired side differs from the current position.

  • Strategy engines generate BUY/SELL/NEUTRAL signals with metadata (EMA, ATR, stop-loss, trailing stop)
  • Instance runner polls each engine at configurable intervals, executes IOC limit orders with 0.5% slippage
  • Backtest runner replays historical candles through the same strategy logic with trailing stop simulation, dynamic mintick, and fee calculation
  • Monitoring system scores engine performance, recommends rotations, and triggers alerts on drawdown or loss streaks

Strategies

Three strategies are currently implemented, each a faithful translation of TradingView PineScript:

  • v1.3 — Scalp strategy with EMA crossover, ATR-based stop-loss, and trailing stop
  • v1 — Swing strategy for longer timeframes (1h+)
  • v6.1 — Dynamic risk multiplier with drawdown protection and pin bar detection

Backtest Results

FARTCOIN v1.3 15m over 30 days: +116.68% return, 240 trades, 90.4% win rate, profit factor 4.48, max drawdown 15.46%. Verified against TradingView with ~26% of TV results (expected gap due to OHLC vs tick-level trailing stop evaluation).

Tech Stack

Python, FastAPI, HyperLiquid Python SDK, SQLAlchemy, Jinja2. PWA-capable with offline shell caching. All secrets in environment variables only — never committed or logged.

Frequently Asked Questions

What is PULS-R?

PULS-R is a strategy execution engine for HyperLiquid perpetuals. It runs PineScript-derived strategies, backtests them against historical data, and executes trades live on HyperLiquid.

How accurate are the backtests?

Backtests use OHLC candle data with trailing stop simulation. TradingView uses calc_on_every_tick=true which gives intra-candle precision we can't match with OHLC alone. We typically achieve 26-50% of TradingView's results. The gap can be closed with intra-bar tick simulation (planned).

Is this safe for live trading?

Yes, with safeguards. DRY_RUN mode is the default. Live trading requires explicitly setting DRY_RUN=false. Per-engine credentials are Fernet-encrypted. All write endpoints require authentication. Kill switches are available at global, per-instance, and withdrawal levels.

How do I connect my wallet?

We recommend using MetaMask. Any wallet that connects to the HyperLiquid DEX will work the same way — PULS-R uses the same signing keys. You can use your existing HL wallet by importing your private key into the engine settings. The key is encrypted with Fernet and never logged or committed. Alternatively, generate a dedicated trading wallet via MetaMask, fund it with USDC on HyperLiquid, and import that key for per-engine credentials.

What HyperLiquid assets are supported?

Any perpetual listed on HyperLiquid. The engine detects mintick per asset from candle data. Tested with FARTCOIN, WIF, HYPE, SOL, ETH, BTC. The HL API has a 60-day candle history limit and 5,000 bars per request.

Can I add my own strategy?

Yes. Create a Python file in engine/ implementing generate_signals() returning {direction, signal, metadata}. Register it in engine/registry.py. The backtest runner and live instance runner will use it automatically.

Does the AI chat execute trades?

No. The LLM chat is advisory only. It can analyze performance, compare strategies, and suggest parameters, but it cannot execute trades, modify engines, or access credentials.

What's the difference from TradingView?

TradingView is a charting platform with PineScript indicators. PULS-R is an execution engine that translates PineScript strategies into Python, runs them live on HyperLiquid, and manages a fleet of independent strategy instances with monitoring, alerts, and withdrawal management.
